# Эволюция административной сетки Западной Сибири — тестовый просмотрщик v2

Лёгкий статический проект для GitHub Pages: Leaflet + стандартизированные GeoJSON-слои.

## Что изменено в v2

- Каждый исходный GeoJSON нормализован: добавлены единые поля `name`, `admin_parent`, `unit_type`, `year`, `start_year`, `end_year`, `population`, `confidence`.
- Все исходные поля сохранены с префиксом `orig_`, чтобы не потерять первичную структуру.
- Формулировки вида «спорная принадлежность», «неясная принадлежность», «неясные границы», «неустойчивый контроль», «спорный контроль» вынесены в единые поля `uncertainty_*`.
- Спорные/неясные/неустойчивые полигоны показываются штриховкой без внешнего контура: `render_hatch = 1`, `render_outline = none`.
- Таймлайн сделан как радиосписок срезов: активен один слой, слои не накапливаются по мере движения шкалы.
- Опционально показывается только один предыдущий срез пунктиром.

## Запуск локально

```bash
python -m http.server 8000
```

Открыть:

```text
http://localhost:8000
```

## Публикация на GitHub Pages

В настройках репозитория:

```text
Settings → Pages → Deploy from a branch → main → /root → Save
```

## Повторная сборка

Скрипт `scripts/rebuild_standardized.py` показывает логику нормализации. В текущем архиве данные уже собраны в `data/layers/`.
