# Эволюция административной сетки Западной Сибири

Лёгкий статический проект для GitHub Pages: GeoJSON-слои открываются по одному через таймслайдер, можно смотреть эволюцию административной сетки, предыдущий срез серым пунктиром, атрибутивную структуру и таблицу объектов.

## Что внутри

```text
index.html              # страница приложения
css/styles.css          # оформление
js/app.js               # логика карты, таймслайдера и таблиц
data/manifest.json      # список слоёв, годы, схемы атрибутов, bbox
data/layers/*.geojson   # очищенные GeoJSON с короткими именами файлов
scripts/rebuild_from_zip.py  # пересборка data/ из нового ZIP с GeoJSON
```

По умолчанию таймлайн показывает только административные границы. В выпадающем списке можно переключить режим на все слои, статистику, остроги/пункты или буферы.

## Быстрый локальный запуск

Открывать `index.html` двойным кликом не стоит: браузер может заблокировать загрузку локальных GeoJSON. Нужен простой локальный сервер.

```bash
cd admin-evolution-west-siberia
python -m http.server 8000
```

Потом открыть:

```text
http://localhost:8000
```

## Публикация на GitHub Pages

1. Создать пустой репозиторий на GitHub.
2. Залить содержимое этой папки в корень репозитория.
3. В настройках репозитория открыть **Settings → Pages**.
4. Включить публикацию из ветки `main`, папка `/root`.
5. Через минуту-две страница будет доступна как GitHub Pages-сайт.

## Обновление данных

Если снова выгрузишь слои из ArcGIS Pro в ZIP, положи ZIP в корень проекта и выполни:

```bash
python scripts/rebuild_from_zip.py GeoJSON_Ocean_5M.zip
```

Скрипт:

- берёт из архива только `*.geojson`;
- декодирует имена вида `#U0413#U0440...`;
- переименовывает файлы в короткие безопасные имена;
- строит `data/manifest.json` со списком полей, типами, примерами значений, количеством объектов и bbox;
- не трогает `index.html`, `css/` и `js/`.

## Примечания по данным

Это просмотрщик, а не система версионирования границ. Он честно показывает каждый GeoJSON как отдельный временной срез. Если слой называется `1782–1797` или `после 1726`, в таймлайне он сортируется по первому найденному году, а подпись сохраняет диапазон/оговорку.

Для тяжёлых будущих наборов лучше перейти на векторные тайлы или PMTiles, но для текущего объёма GeoJSON нормален и проще для GitHub Pages.
